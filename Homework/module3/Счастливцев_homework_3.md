# Звіт: Керування процесами Linux

---

## Завдання 1 — Огляд активних процесів

### 1.1 Список усіх процесів (`ps aux`)

```
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1 11.0  0.1  15136  5416 ?        Sl   13:20   0:00 /process_api --firecracker-init ...
root         2  0.0  0.0      0     0 ?        S    13:20   0:00 [kthreadd]
root       491 48.0  0.8 1939528 34732 ?       Sl   13:20   0:00 /opt/rclone/rclone-filestore multimount ...
...
```

> `ps aux` виводить усі процеси системи з деталями: користувач, PID, завантаження CPU та RAM, розмір пам'яті, стан, час запуску і команду.

---

### 1.2 Процес із найбільшим споживанням RAM

```bash
ps aux --sort=-%mem | head -15
```

| PID | %CPU | %MEM | COMMAND |
|-----|------|------|---------|
| 491 | 46.2 | 0.8% | `/opt/rclone/rclone-filestore multimount` |
| 1   | 11.0 | 0.1% | `/process_api --firecracker-init` |

**Висновок:** найбільше RAM споживає процес `rclone-filestore` (PID 491) — файловий сервіс монтування.

---

### 1.3 PID поточної оболонки

```bash
echo "PID: $$"
ps -p $$ -o pid,ppid,comm,args
```

**Результат:**

```
PID поточного процесу ($$): 516
  PID  PPID COMMAND
  516     1 sh
```

Поточна оболонка — `/bin/sh`, PID = **516**, батьківський процес — PID 1 (`process_api`).

## Завдання 2 — Робота у фоні та керування процесами

### 2.1 Запуск команди у фоновому режимі

```bash
sleep 1000 &
```

**Результат:**
```
Процес sleep 1000 запущено з PID: 522
```

Символ `&` в кінці команди запускає процес у фоні, не блокуючи термінал.

---

### 2.2 Список фонових завдань

```bash
jobs -l
```

**Результат:**
```
[1] + 522 Running    sleep 1000
```

`jobs -l` показує номер завдання, PID і стан (`Running`, `Stopped` тощо).

---

### 2.3 Повернення процесу на передній план

```bash
fg %1
```

Команда `fg %1` переводить завдання №1 з фону на передній план. У скрипті демонстрували через `SIGSTOP` / `SIGCONT`:

```bash
kill -STOP 522   # зупинити
kill -CONT 522   # відновити
```

---

### 2.4 Зупинка та примусове завершення

```bash
kill -15 522   # SIGTERM — коректне завершення (дає процесу час прибрати за собою)
kill -9  522   # SIGKILL — примусове знищення (не можна ігнорувати)
```

**Результат:**
```
Надіслано SIGTERM до PID 522
Процес вже завершено після SIGTERM
```

> Різниця між сигналами:
> - **SIGTERM (15)** — «м'яке» завершення, процес може зберегти стан
> - **SIGKILL (9)** — негайне знищення, ядро завершує процес без попередження

---

### 2.5 Запуск через `nohup`

```bash
nohup sleep 500 > /tmp/nohup_test.out 2>&1 &
```

**Результат:**
```
nohup sleep 500 запущено з PID: 524
Вивід збережено у: /tmp/nohup_test.out
Процес активний!
```

`nohup` ігнорує сигнал `SIGHUP`, тому процес продовжує працювати навіть після закриття терміналу.

## Завдання 3 — Пріоритети та обмеження

### 3.1 Запуск з підвищеним значенням `nice`

```bash
nice -n 15 sleep 300 &
```

**Результат:**
```
PID: 527  NI: 15  PRI: 4  COMMAND: sleep
```

> Значення `nice` від **-20** (найвищий пріоритет) до **+19** (найнижчий). Значення `15` — нижчий пріоритет, процес отримає менше часу CPU при конкуренції.

---

### 3.2 Зміна пріоритету запущеного процесу (`renice`)

```bash
renice -n 10 -p 527
```

**Результат:**
```
527 (process ID) old priority 15, new priority 10
  PID  NI PRI COMMAND
  527  10   9 sleep
```

Пріоритет змінено з NI=15 на NI=10 — процес став трохи «важливішим» для планувальника.

---

### 3.3 Обмеження ресурсів (`ulimit -a`)

```bash
ulimit -a
```

| Параметр | Значення |
|----------|----------|
| Час CPU | необмежено |
| Розмір файлу | необмежено |
| Розмір даних | необмежено |
| Розмір стека | **8192 КБ** |
| Core dump | 0 (вимкнено) |
| Пам'ять | необмежено |
| Locked memory | 8192 КБ |
| Макс. процесів | **15 955** |
| Відкритих файлів | **1024** |
| Віртуальна пам'ять | необмежено |
| RT-пріоритет | 0 |

## Завдання 4 — Моніторинг ресурсів

### 4.1 Дисковий простір (`df -h`)

```bash
df -h
```

| Файлова система | Розмір | Використано | Вільно | % |
|----------------|--------|-------------|--------|---|
| `/dev/vda` (корінь) | 252 ГБ | 8.6 ГБ | 10 ГБ | 47% |
| `tmpfs` (/dev/shm) | 2.0 ГБ | 0 | 2.0 ГБ | 0% |
| `rclone` (outputs) | 1.0 ПБ | 0 | 1.0 ПБ | 0% |

Прапор `-h` (human-readable) виводить розміри у зручному форматі (КБ/МБ/ГБ).

---

| **Swap** | 0 | 0 | 0 | — | — |
